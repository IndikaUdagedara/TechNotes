ThinkPad T14s Gen4 internal mic stopped working (no audio on Google Meet). Root cause: ALSA Capture control on card 1 (ALC257 codec) was muted and at 0% volume. The mic mute LED (/sys/class/leds/platform::micmute/brightness) was also set to 1 (active). Fix: unmuted Capture and set to 70%, set Mic Boost to 1 (33%, +10dB) using amixer -c 1. Audio stack context: ALSA is the kernel driver layer; PulseAudio and PipeWire are user-space servers built on top of it. This system runs PipeWire (latest). PipeWire does not manage hardware-level ALSA mixer controls like Capture and Mic Boost. To persist the fix across reboots in NixOS, the idiomatic solution is hardware.alsa.enablePersistence = true in the host config (hosts/thinkpad/default.nix). This installs an alsa-store systemd service that restores /var/lib/alsa/asound.state on boot and saves it on shutdown, plus a udev rule for hotplug. It is independent of hardware.alsa.enable and does NOT disable PipeWire. After adding the NixOS option, run sudo alsactl store once to capture the desired mixer state, then sudo nixos-rebuild switch.
